TryHackMe Room Report: Mr. Robot

Author: Luyanda

Date: 31 March 2026

1. Room Overview

The Mr. Robot room is a beginner-friendly TryHackMe room inspired by the TV series. It focuses on basic Linux commands, enumeration, and gaining initial access through web vulnerabilities.

2. Objectives

  • Understand basic Linux commands and file navigation.
  • Practice enumeration using Nmap and Dirb/Gobuster.
  • Retrieve user and root flags by exploiting misconfigurations.
  • Learn to document steps for penetration testing.

3. Walkthrough

  1. Scanning: Used nmap to identify open ports (typically SSH and HTTP).
  2. Enumeration: Explored directories using Gobuster to find hidden files or web panels.
  3. Exploitation: Found credentials in files or via default passwords; used SSH to access the machine.
  4. Privilege Escalation: Checked system for vulnerable SUID files, misconfigured scripts, and readable logs to gain root access.
  5. Flag Collection: Captured user.txt and root.txt flags and documented commands used.

4. Challenges & Solutions

Challenge: Finding the hidden credentials for SSH login.

Solution: Enumerated web directories using Gobuster, discovered a backup file containing credentials, then logged in with SSH.

5. Tools Used

  • Nmap – Port scanning
  • Gobuster – Directory enumeration
  • SSH – Remote login
  • Linux terminal commands (ls, cat, find)

6. Lessons Learned

  • Importance of enumeration for finding hidden files and directories.
  • Basic Linux command proficiency is essential for penetration testing.
  • Always check for misconfigured files or default credentials.
  • Documenting every step is crucial for reports and reproducibility.

7. Conclusion

The Mr. Robot TryHackMe room provides a hands-on introduction to Linux enumeration, SSH access, and privilege escalation. It reinforces core penetration testing skills in a beginner-friendly environment and demonstrates how small oversights can be exploited by attackers.
